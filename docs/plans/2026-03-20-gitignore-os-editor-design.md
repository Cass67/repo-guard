# .gitignore OS and editor coverage design

## Goal

Extend the root and template `.gitignore` files so repositories generated by `repo-guard` ignore common OS metadata and editor workspace junk in addition to secret-bearing files.

## Approach

Add one clearly labeled block to both tracked `.gitignore` files.

The new block should:
- include broadly safe macOS, Windows, Linux desktop, and common editor artifacts
- avoid language-specific build outputs and repo-specific generated files
- preserve the current secret-first structure and stay easy to review

## Planned entries

- macOS: `.DS_Store`, `.AppleDouble`, `.LSOverride`, `._*`, `.Spotlight-V100/`, `.Trashes/`
- Windows: `Thumbs.db`, `Desktop.ini`, `$RECYCLE.BIN/`
- Linux desktop trash: `.directory`
- Editors and temp files: `.vscode/`, `.idea/`, `*.swp`, `*.swo`, `*~`

## Verification

- confirm both `.gitignore` files contain the new block
- keep existing secret-related entries unchanged
